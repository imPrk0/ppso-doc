环境变量 ​

如果你不想以自行修改文件的方式去编辑环境变量配置，可以使用我们提供的在线可视化配置工具。

环境变量介绍 ​

如果你想详细了解每一个配置项是做什么的，或想要手写环境变量，那么恭喜你！下面的内容就是为你量身打造的！

ADMIN_PATH ​

系统管理员进入目录

  • 非必填
  • 默认：prk_admin

如果你配置成 admin 那么你进入管理后台的地址就是 https://www.example.com/admin/。

当然，该配置项支持多级目录，如果你愿意的话，配置成下面这样也是可以的，

bash
ADMIN_PATH = "a/b/c/d/e/f/g/h/i/j/k/l/m/n/o/p/q/r/s/t/u/v/w/x/y/z"

那么你进入后台的地址就是 https://www.example.com/a/b/c/d/e/f/g/h/i/j/k/l/m/n/o/p/q/r/s/t/u/v/w/x/y/z/。

这样做的目的是有效防止恶意扫描后台目录。

温馨提示

配置此项推荐配合防火墙，可以有效防止后台目录被扫。

DOMAIN_NAME ​

如果配置，则强行绑定域名；如果不是该配置的域名将强行跳转至该域名。

DB_CONNECTION ​

数据库连接方式：

数据库值
MySQLmysql
SQLitesqlite
PostgreSQLpgsql
SQL Serversqlsrv

DB_HOST ​

数据库连接地址，默认 127.0.0.1。

DB_PORT ​

数据库连接端口。

数据库默认端口
MySQL3306

DB_DATABASE ​

数据库名或 SQLite 数据库文件绝对或相对路径。

DB_USERNAME ​

数据库用户名，通常默认为 root。

DB_PASSWORD ​

数据库用户密码或连接密码。

APP_DEBUG 开发环境 ​

开启后将进入调试模式，页面会暴露出敏感内容，切勿实际开启！默认关闭即可！
